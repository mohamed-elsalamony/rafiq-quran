# خادم المساعد الذكي لتطبيق رفيق القرآن (Rafiq AI Backend)

هذا هو خادم الـ Backend الوسيط المسؤول عن الاتصال بنموذج ذكاء اصطناعي (Gemini) لتوفير ردود ذكية، إسلامية وعامة، دقيقة مع حماية مفتاح الـ API بالكامل ومنع تسريبه أو تخزينه في تطبيق الهاتف.

---

## الميزات الأساسية
1. **أمان تام**: لا يتم تضمين مفتاح الـ API داخل تطبيق الفلاتر أو الـ APK، بل يظل مخزناً بشكل آمن على السيرفر في ملف البيئة `.env`.
2. **توجيهات فقهية صارمة**: يفرض السيرفر توجيهات نظام (System Instructions) صارمة على Gemini تمنعه من اختلاق آيات قرآنية أو أحاديث أو إصدار فتاوى فقهية فردية دون مرجعية.
3. **أداء وسرعة**: تصفية فورية وسريعة للشبكة والتحول التلقائي بين الوضع المتصل والوضع غير المتصل.

---

## متطلبات التشغيل
- تثبيت [Node.js](https://nodejs.org/) (إصدار 18 فما فوق).

---

## طريقة الإعداد والتشغيل المحلي

### 1. تثبيت الاعتمادات (Dependencies)
افتح الطرفية (Terminal) في مجلد الـ backend وقم بتشغيل الأمر التالي:
```bash
npm install
```

### 2. إعداد مفتاح الـ API
قم بنسخ ملف الإعداد `.env.example` إلى ملف جديد باسم `.env`:
```bash
cp .env.example .env
```
افتح ملف `.env` وقم بوضع مفتاح Gemini الخاص بك في المتغير `GEMINI_API_KEY`:
```env
PORT=3000
GEMINI_API_KEY=AIzaSy...
GEMINI_MODEL=gemini-2.5-flash
```

### 3. تشغيل السيرفر
لتشغيل السيرفر في وضع التطوير (مع التحديث التلقائي عند التعديل):
```bash
npm run dev
```

لتشغيله في وضع الإنتاج:
```bash
npm start
```

سيبدأ السيرفر في الاستماع على المنفذ `3000`. يمكنك التحقق من تشغيله بفتح الرابط التالي في المتصفح:
`http://localhost:3000/health`

## الربط مع تطبيق الهاتف (Flutter)

عند تشغيل التطبيق في بيئات التطوير المختلفة، قم بتهيئة عنوان السيرفر من خلال **شاشة الإعدادات** بالتطبيق كالتالي:

1. **محاكي الأندرويد (Android Emulator)**: استخدم العنوان الافتراضي المدمج:
   `http://10.0.2.2:3000`
2. **محاكي الـ iOS أو المتصفح (iOS Simulator / Web)**: استخدم:
   `http://localhost:3000`
3. **هاتف حقيقي (Real Device)**: تأكد من أن الهاتف والكمبيوتر متصلان بنفس شبكة الواي فاي، ثم استخدم عنوان الآي بي الخاص بجهاز الكمبيوتر الخاص بك، مثال:
   `http://192.168.1.15:3000`

## النشر في الإنتاج (Production Deployment)

يمكنك نشر هذا الخادم مجاناً وبسهولة على منصات السحاب مثل:
- **Render** ([render.com](https://render.com/))
- **Railway** ([railway.app](https://railway.app/))

### خطوات النشر على Render:
1. ارفع مجلد الـ backend إلى مستودع GitHub (مستودع خاص أو عام).
2. أنشئ حساباً على Render وقم بربطه بـ GitHub.
3. أنشئ خدمة جديدة من نوع **Web Service** واختر مستودع الـ backend.
4. في إعدادات البيئة (Environment Variables)، أضف المتغير `GEMINI_API_KEY` وقيمته.
5. سيمنحك Render رابطاً آمناً يبدأ بـ `https://` (مثل: `https://rafiq-ai.onrender.com`).
6. انسخ هذا الرابط وضعه في إعدادات التطبيق!
