# دليل إعداد ونشر Firebase & Cloud Functions & FCM لـ "رفيق القرآن"

يوضح هذا الدليل الخطوات اللازمة لإعداد ونشر البنية الخلفية السحابية لـ Firebase Cloud Functions (والتي تعمل كـ Proxy للمساعد الذكي) بالإضافة إلى تهيئة إشعارات Firebase Cloud Messaging (FCM) على التطبيق.

---

## الجزء الأول: تهيئة مشروع Firebase وتطبيق الهاتف (FCM)

لكي يعمل نظام Firebase والتحليلات والإشعارات السحابية (FCM) على التطبيق:

### الخطوة 1: إنشاء مشروع على Firebase
1. انتقل إلى [Firebase Console](https://console.firebase.google.com/).
2. اضغط على **Add Project** وأنشئ مشروعاً جديداً باسم `rafiq-quran` (أو أي اسم تفضله).

### الخطوة 2: إضافة التطبيق للمشروع (Android & iOS)
1. **لتطبيق أندرويد (Android):**
   * اضغط على أيقونة Android في واجهة مشروع Firebase.
   * أدخل اسم الحزمة (Package Name): `com.mohamedelsalamony.rafiq_quran` (أو كما هو محدد في ملف `android/app/build.gradle`).
   * قم بتحميل ملف `google-services.json`.
   * انسخ الملف وضعه في المسار التالي داخل المشروع: `android/app/google-services.json`.
2. **لتطبيق آي أو إس (iOS):**
   * اضغط على **Add App** ثم اختر iOS.
   * أدخل الـ Bundle ID الخاص بالتطبيق.
   * قم بتحميل ملف `GoogleService-Info.plist`.
   * انسخ الملف وضعه في مجلد `ios/Runner/` عن طريق Xcode.

> [!NOTE]
> **ملاحظة حول أمان التشغيل:**
> تم تصميم الكود ليتفادى الانهيار تماماً في حال عدم وجود هذه الملفات (مثل `google-services.json`). سيعمل التطبيق بشكل طبيعي ويتجاوز تهيئة Firebase مع كتابة تحذير في سجل الأخطاء (Log) دون تعطل التطبيق عند العميل.

## الجزء الثاني: إعداد ونشر خادم المساعد الذكي (Cloud Functions)

تم بناء وظيفة سحابية مبسطة (Proxy) تستلم رسالة المستخدم وترسلها لـ Gemini API للحفاظ على سرية مفتاح الـ API وتجنب استدعائه مباشرة من التطبيق.

### الخطوة 1: تثبيت أدوات Firebase CLI
افتح Terminal في جهازك وقم بتشغيل:
```bash
npm install -g firebase-tools
```

### الخطوة 2: تسجيل الدخول لـ Firebase
شغّل الأمر التالي لتسجيل الدخول عبر المتصفح:
```bash
firebase login
```

### الخطوة 3: ربط المجلد بمشروعك
في المجلد الرئيسي للمشروع (حيث يوجد ملف `firebase.json` ومجلد `functions`)، شغّل:
```bash
firebase use --add
```
اختر مشروعك الذي أنشأته من القائمة، وسمِّ الاسم المستعار بـ `default`.

### الخطوة 4: إعداد مفاتيح البيئة للذكاء الاصطناعي
1. افتح مجلد `functions` وقم بإنشاء ملف باسم `.env` (إذا لم يكن موجوداً).
2. أضف الكود التالي واكتب مفتاح الـ API الخاص بك من Google AI Studio:
```env
GEMINI_API_KEY=AIzaSy...اكتب_مفتاحك_هنا...
GEMINI_MODEL=gemini-2.5-flash
```

### الخطوة 5: تثبيت الحزم ونشر الـ Functions
شغّل الأوامر التالية من المجلد الرئيسي للمشروع:
```bash
cd functions
npm install
cd ..
firebase deploy --only functions
```

## الجزء الثالث: ربط التطبيق واختبار الإشعارات السحابية

### 1. ربط المساعد الذكي بالتطبيق
* بعد النشر بنجاح، ستطبع لك Firebase رابط النهاية (Endpoint URL) مثل: `https://api-xxxxxx.run.app`.
* افتح التطبيق -> اذهب إلى **شاشة الإعدادات** -> قسم **إدارة حالة المساعد (AI Backend)**.
* الصق الرابط في حقل **عنوان سيرفر الـ Backend** ثم اضغط **حفظ وتفعيل**.
* سيتحول شريط الحالة فوراً إلى **متصل بالمساعد الذكي (Online)** باللون الأخضر.

### 2. اختبار الإشعارات الفورية والسحابية (FCM)
* في **شاشة الإعدادات**، ستجد قسماً جديداً باسم **خدمة الإشعارات السحابية (FCM)**.
* في حال نجاح اتصال التطبيق بـ Firebase، ستظهر حالة الخدمة **نشط ومتصل بـ Firebase** باللون الأخضر وسيظهر الـ **FCM Token** الخاص بجهازك.
* يمكنك الضغط على أيقونة النسخ المجاورة للـ Token لنسخه.
* لاستقبال الإشعارات السحابية الفورية في الخلفية أو الواجهة، يمكنك إرسال إشعار تجريبي من Firebase Console باستخدام الـ Token المنسوخ للتحقق من وصول التنبيهات ونغمة الأذان بنجاح.
* لاختبار النظام المحلي للإشعارات دون أي تأخير، اضغط على **إرسال إشعار تجريبي فوري** لتصلك رسالة اختبارية في غضون جزء من الثانية للتأكد من الصلاحيات وسلامة الخدمة.
